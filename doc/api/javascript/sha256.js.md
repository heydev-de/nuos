## SHA-256 Hashing Utility

Pure JavaScript implementation of the SHA-256 cryptographic hash function. This utility provides a client-side mechanism for generating SHA-256 hashes without external dependencies, ensuring data integrity and security for sensitive operations like password hashing or checksum verification.

---

### `sha256(str)`

Generates a SHA-256 hash of the input string.

#### Parameters

| Name | Type   | Description                          |
|------|--------|--------------------------------------|
| str  | string | Input string to be hashed. Must be a UTF-8 encoded string without multibyte characters (only ASCII-compatible). |

#### Return Value

| Type   | Description                          |
|--------|--------------------------------------|
| string | 64-character lowercase hexadecimal string representing the SHA-256 hash of the input. |

#### Inner Mechanisms

1. **Preprocessing**:
   - Appends a single `0x80` byte to the input string.
   - Pads the string with null bytes (`\0`) to ensure its length is a multiple of 64 bytes (512 bits), with the last 8 bytes reserved for the original message length in bits.

2. **Message Schedule**:
   - Converts the input string into an array of 32-bit words (`w`).
   - Extends the 16-word block into a 64-word schedule using bitwise operations (`rot`, XOR, and right-shift).

3. **Compression**:
   - Initializes the hash state with 8 predefined 32-bit constants (first 32 bits of the fractional parts of the square roots of the first 8 primes).
   - Processes each 512-bit block through 64 rounds of bitwise operations, modular additions, and rotations, updating the hash state iteratively.

4. **Finalization**:
   - Combines the intermediate hash values into a single 256-bit result.
   - Converts the result into a 64-character hexadecimal string.

#### Usage Context

- **Client-Side Security**: Ideal for hashing sensitive data (e.g., passwords) before transmission to the server, reducing exposure of raw data.
- **Data Integrity**: Useful for generating checksums or verifying file integrity in environments where server-side processing is unavailable.
- **Compatibility**: Designed for lightweight, dependency-free use in browsers or Node.js environments.

#### Example Usage

```javascript
const input = "Hello, NUOS!";
const hash = sha256(input); // "dffd6021bb2bd5b0af676290809ec3a53191dd81c7f70a4b28688a362182986f"
console.log(hash);
```

#### Limitations

- **Multibyte Characters**: The implementation assumes ASCII-compatible input. For multibyte strings (e.g., UTF-8), preprocess the input using `TextEncoder` or equivalent to ensure correct byte representation.
- **Performance**: Pure JavaScript implementations are slower than native alternatives (e.g., Web Crypto API). For performance-critical applications, consider using platform-native APIs where available.
